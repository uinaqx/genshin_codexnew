# 官方外观配色规范

v2.0 的主题是配色数据，不是渲染器注入包。

## 文件结构

```text
themes/<theme-id>/
├── palette.json
└── art.webp        # 可选，仅作为仓库预览素材，不安装到应用
```

`palette.json` 必须通过 `schema/palette.schema.json`，并包含：

- `id`、`name`、`version` 和浅色/深色模式；
- 强调色、背景色、前景色、辅助色和表面色；
- 四个值必须为 `false` 的安全声明；
- 指向官方“设置 → 外观”的应用说明。

## 安全限制

主题不得包含或要求：

- 调试端口、CDP 或脚本注入；
- 修改应用包、签名资源或启动参数；
- watcher、自动重启或登录启动项；
- 写入未公开的内部配置；
- 清理缓存、账号、对话或项目数据。

## 视觉验收

- 正文前景色与背景色对比度至少达到 WCAG AA，建议 AAA；
- 白色按钮文字与强调色对比度至少 4.5:1；
- 预览图不得包含私人项目名、账号信息或真实对话；
- 素材必须有可说明的创作或授权来源；
- 实际效果必须标注为官方外观能力，不得把概念预览说成自动注入后的截图。

运行：

```bash
node tests/test-palette.mjs
```
