# v1.x Windows 启动故障复盘

日期：2026-07-20
状态：v2.0 已移除相关实现

## 现象

在 Microsoft Store 版 ChatGPT/Codex 更新后，v1.x 尝试启用可莉皮肤会失败。部分机器随后长期停在启动标志画面；选择恢复官方皮肤后也可能继续停留。

## 根因

这不是单一的网络或电脑问题。旧版设计与新版桌面应用的启动和安全策略不兼容，并且失败处理扩大了影响范围。

1. 旧版依赖 `--remote-debugging-port` 启动参数和 CDP 注入。新版 Store 应用会拒绝或忽略这种启动方式。
2. 启用流程先写 `~/.codex/config.toml`，随后才验证应用能否通过调试方式启动。启动失败时，配置已经发生变化。
3. watcher 会监视并重新启动皮肤流程。异常状态下可能重复干预正常启动。
4. v1.1.3 安装完成后会自动执行紧急恢复，而不是等待用户明确选择。
5. 紧急恢复包含 `Reset-AppxPackage`、`Add-AppxPackage`、MSIX 配置目录移动和账号状态处理。这些操作超出了皮肤工具应有的范围，并可能要求重新登录。
6. 当时的自动测试把上述恢复行为当成“必须存在”的功能，无法阻止高风险代码发布。

## v2.0 修复

- 删除所有 Windows/macOS 调试端口启动、CDP 注入、watcher 和自动重启代码；
- 删除应用包重置、重新注册、缓存清理和账号清理代码；
- 安装程序不再执行任何恢复或换肤动作；
- 卸载程序不再运行恢复钩子，也不删除用户状态目录；
- v2.0 使用新的安装标识和目录，避免继承 v1.x 卸载器中保存的删除规则；
- 清理脚本仅处理已知的 v1.x 精确目标，并在改配置前备份；
- 诊断只报告版本和布尔状态，不读取凭据、聊天或项目内容；
- 新增反向安全测试，明确禁止高风险关键字与旧版注入文件；
- 配色改为通过官方“设置 → 外观”手动确认。

## 数据保留原则

v2.0 的清理脚本不会删除以下内容：

- `.codex\sessions`；
- `.codex\archived_sessions`；
- `auth.json` 和 `cap_sid`；
- ChatGPT/Codex 官方应用缓存；
- 任何项目目录；
- Microsoft Store 应用包。

如果当前官方客户端已经可以启动，不需要执行额外重置。只有安全中心明确检测到 v1.x 残留时，才按需运行清理。

## 后续兼容策略

主分支不再根据未公开的内部实现做桌面界面注入。只有 OpenAI 提供公开、稳定、可测试且不改变应用启动链路的扩展接口后，才会重新评估人物背景等高级效果。
